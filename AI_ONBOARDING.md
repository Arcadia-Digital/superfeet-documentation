## Methodology Principles

**Business-First Approach:**
- Write for business users, not just developers
- Balance technical accuracy with business clarity
- Progressive disclosure (high-level → details)
- Use case-driven organization

**Evidence-Based Documentation:**
- Base documentation on actual codebase analysis
- Document only integrations with theme code evidence
- Verify all documentation against source material
- Use real data exports, not assumptions

**Navigation Documentation Standards (Required):**
- Menu-to-section mapping table
- Comprehensive global element documentation
- Code verification against theme files
- Desktop vs. mobile behavior documentation
