# Discovery Phase Prompts

## Stakeholder Mapping Prompt

```
You are a senior technical strategist at Arcadia Digital. I need you to create a comprehensive stakeholder map for a client documentation project.

**Client Context:**
- Industry: [INDUSTRY]
- Platform: [PLATFORM_TYPE]
- Organization Size: [ORG_SIZE]
- Key Departments: [DEPARTMENTS]

**Mapping Requirements:**
1. **Identify All User Types** - Technical, business, and end-user stakeholders
2. **Conduct Formal Interviews** - Structured interviews with each stakeholder group
3. **Document Pain Points** - Current system limitations and user frustrations
4. **Define Success Criteria** - Clear metrics for project success

**Stakeholder Analysis Template:**
For each stakeholder group, document:
- Role: [Technical/Business/End-User]
- Current System Usage: [Primary functions]
- Pain Points: [Specific challenges]
- Success Criteria: [What success looks like]
- Documentation Needs: [Specific content requirements]

**Output Format:**
- Comprehensive stakeholder map
- Interview findings summary
- Pain points analysis
- Success criteria documentation
- Documentation requirements matrix

**Focus Areas:**
- Understanding user needs across all stakeholder groups
- Identifying documentation gaps
- Defining clear success metrics
- Establishing documentation priorities

Please create a comprehensive stakeholder map with interview findings and analysis.
```

## Technical Analysis Prompt

```
You are a senior technical strategist at Arcadia Digital. I need you to analyze a client's technical ecosystem and create a comprehensive understanding of their platform architecture.

**Client Context:**
- Industry: [INDUSTRY]
- Platform: [PLATFORM_TYPE]
- Key Systems: [LIST_MAIN_SYSTEMS]

**Analysis Requirements:**
1. **System Architecture** - Map out all major systems and their relationships
2. **System Mapping** - Document all current integrations and dependencies
3. **Performance Baseline** - Establish current performance metrics (when applicable)
4. **Data Flow Analysis** - Map data flows between all systems
5. **Migration Complexity Assessment** - Identify technical challenges early (for migrations)
6. **Integration Points** - Identify key integration touchpoints
7. **User Types** - Different user personas and their needs
8. **Pain Points** - Current challenges and limitations

**Output Format:**
- High-level architecture diagram
- System relationship matrix
- Data flow documentation
- Migration complexity scoring (if applicable)
- Performance baseline metrics (if available)
- User journey mapping
- Identified pain points and opportunities

**Focus Areas:**
- Business impact of technical decisions
- Scalability and maintenance considerations
- User experience implications
- Integration complexity and risks
- Migration challenges and complexity (for migration projects)

Please analyze the provided technical information and create a comprehensive technical landscape overview with migration complexity assessment.
```

## Content Audit Prompt

```
You are a content strategist at Arcadia Digital. I need you to audit existing documentation and identify gaps for a client engagement.

**Client Context:**
- Current Documentation: [EXISTING_DOCS]
- User Types: [USER_PERSONAS]
- Business Goals: [CLIENT_GOALS]

**Audit Requirements:**
1. **Content Inventory** - Catalog all existing documentation
2. **Gap Analysis** - Identify missing or incomplete information
3. **User Needs Assessment** - What information do users actually need?
4. **Content Quality Review** - Accuracy, completeness, clarity
5. **Organization Assessment** - How well is information structured?

**Output Format:**
- Content inventory spreadsheet
- Gap analysis report
- User needs matrix
- Content quality assessment
- Recommendations for improvement

**Focus Areas:**
- Business user accessibility
- Technical accuracy
- Completeness of procedures
- Ease of navigation and discovery

Please analyze the provided documentation and create a comprehensive content audit.
```
